## Firebase Emulator + 규칙 자동 테스트

섹션 M의 RBAC 시나리오를 수동 체크 없이 자동 검증하는 방식입니다.

### 1회성 준비 — Java JDK 설치 (필수)
Firebase Emulator Suite는 Firestore/Storage 에뮬레이터를 Java로 실행해요.

**Windows 가장 빠른 방법:**
```powershell
winget install Microsoft.OpenJDK.21
```

설치 후 새 터미널에서 확인:
```bash
java -version
# openjdk version "21.x.x" ... 가 뜨면 OK
```

### 에뮬레이터 실행 + 테스트

**터미널 A** — 에뮬레이터 기동 (계속 떠있어야 함):
```bash
npm run emulators
```
- http://localhost:4000 — Emulator UI (데이터 브라우저, Rules Playground)
- Auth: 9099 / Firestore: 8080 / Storage: 9199

**터미널 B** — 규칙 테스트 실행:
```bash
npm run test:rules
```

**또는 CI 한 방 모드** (에뮬레이터 자동 기동 + 테스트 + 종료):
```bash
npm run test:rules:ci
```

### 테스트 커버리지 (`scripts/test-rules.mjs`)

| 그룹 | 시나리오 | 검증 |
|---|---|---|
| users | 자기 프로필 읽기/타인 읽기/role 자기승격/non-role 업데이트 | 4 |
| private/config | 부모 읽기/자녀 차단/외부인 차단/부모 쓰기/자녀 쓰기 차단 | 5 |
| rewards | 멤버 읽기/외부인 차단/부모 생성/자녀 생성 차단 | 4 |
| categories | 부모 생성/자녀 삭제 차단 | 2 |
| quests | 자녀 체크만 허용/title 차단/points 차단/부모 CRUD/자녀 생성·삭제 차단 | 7 |
| cross-family | 외부인이 children/feed/rewards 접근 차단 | 3 |
| feed (SNS) | 멤버 읽기/작성자 위조 차단/반응 토글/비작성자 text 수정 차단/작성자 삭제/비작성자 삭제 차단/댓글 작성/댓글 위조 차단 | 9 |
| unauthenticated | 익명 users/children 읽기 차단/쓰기 차단 | 3 |

**총 37개 자동 검증** — TEST_SCENARIOS.md 섹션 M의 수동 체크리스트를 완전히 대체합니다.

### 에뮬레이터로 앱 실행 (수동 탐색)
```bash
# 터미널 A
npm run emulators

# 터미널 B
npm run dev:emulator
```
이제 앱은 http://localhost:3000 에서 로컬 에뮬레이터에 연결돼요:
- 진짜 Firebase 프로젝트 건드리지 않음
- 에뮬레이터 UI (http://localhost:4000)에서 쓰기 실시간 관찰
- Auth 에뮬레이터에 가짜 유저 직접 생성 가능
- 종료하면 데이터 전부 초기화

### 아키텍처 메모
- `src/firebase.ts`가 `VITE_USE_EMULATOR=1` 환경변수를 감지해 자동 연결
- `firebase.json`이 포트 + 규칙 파일 경로 정의
- `.firebaserc`는 `kidquest-emulator` 더미 프로젝트 ID 사용 (실 프로젝트 분리)
- 규칙 테스트는 `@firebase/rules-unit-testing`의 `initializeTestEnvironment`로 격리된 Firestore 인스턴스 사용 → 한 번 실행에 병렬 유저 시뮬레이션

## 배포 전 실행 순서

```bash
# 1. 자동 테스트
npm run lint                           # tsc --noEmit
node scripts/unit-tests.mjs            # 24/24 pure logic
npx vite build                         # 코드 스플리팅 확인

# 2. 스모크
npx vite --port=5199 &                 # 백그라운드 실행
SMOKE_URL=http://127.0.0.1:5199/ node scripts/smoke-extended.mjs

# 3. 수동 체크리스트 진행 (위 섹션 A-N)

# 4. 규칙 배포
firebase deploy --only firestore:rules,storage

# 5. 앱 배포
# (기존 배포 플로우 따름)
```
