# 한국어 UX 카피 가이드 (아이 대상)

## 톤 원칙
1. **자녀 UI → 반말 친근체** — "오늘도 화이팅!", "멋진걸!"
2. **부모 UI → 존댓말 정중체** — "퀘스트를 추가했어요", "자녀의 활동을 확인해보세요"
3. **공통** — 짧고 긍정적. 한 문장 15자 내외 선호
4. **이모티콘** — 자녀 화면에 적극 사용 (🌟 🎉 ✨ 💪 🎯). 부모 화면은 절제

## 금지
- ❌ 비교 ("다른 친구들은 다 했는데")
- ❌ 처벌/위협 ("안 하면 혼나요")
- ❌ 부정 프레이밍 ("실패했어요") → "다시 도전해볼까?"
- ❌ 어려운 한자어 ("완수", "이행", "귀하")
- ❌ 영어 전문용어 ("로그인" OK, "authentication" 금지)

## 상황별 문구 예시

### 빈 상태 (자녀)
| 상황 | 좋음 | 나쁨 |
|---|---|---|
| 퀘스트 없음 | "오늘의 퀘스트를 기다리는 중! 🌟" | "퀘스트가 없습니다" |
| 완료한 것 없음 | "첫 퀘스트에 도전해볼까? 💪" | "완료 항목 없음" |
| 인벤토리 빔 | "포인트를 모아서 보상을 받아보자!" | "보유 아이템 0개" |

### 성공 피드백 (자녀)
- "멋져! 🎉"
- "대단한걸! +{points}점"
- "오늘도 한 걸음! ✨"
- "네가 해냈어! 🌟"

### 에러 (자녀)
- "앗, 잠깐 문제가 생겼어. 다시 해볼까?"
- "인터넷 연결을 확인해줘!"
- **금지**: "Error: Network request failed"

### 부모 UI
| 기능 | 문구 |
|---|---|
| 퀘스트 추가 버튼 | "+ 퀘스트 만들기" |
| 보상 관리 | "보상 관리하기" |
| 자녀 진행 현황 | "오늘의 활동" |
| 초대 코드 공유 | "자녀에게 이 코드를 알려주세요" |

## Gemini 프롬프트 톤 일관성
AI 생성 격려 메시지도 위 톤을 유지. 프롬프트에 명시:
```
- 반말로
- 15자 이내
- 이모티콘 1~2개
- 비교/부정 금지
```
자세한 Gemini 사용은 `gemini-quest-ai` 스킬.

## 카테고리 레이블 ([src/types.ts:87](src/types.ts#L87))
- `homework` → **숙제**
- `chore` → **심부름/도움**
- `habit` → **좋은 습관**
- `other` → **기타**

새 카테고리 추가 시 레이블도 반드시 아이 친화적으로.

## 포인트 표기
- 자녀: "20점", "+20" (짧게)
- 부모: "20 포인트" (정식)
- 천 단위 구분: "1,200점"

## 레벨업/보상 문구
- 레벨업: "레벨 {n} 달성! 🎊"
- 보상 구매: "{reward} 획득! 🎁"
- 포인트 부족: "앗, {n}점 더 모아야 해!" (격려 톤, 비난 금지)

## 버튼 레이블
동사 원형 짧게: "완료", "저장", "취소", "추가", "교환", "되돌리기". "~하기" 접미사는 부모 UI에만.
